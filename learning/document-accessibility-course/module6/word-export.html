Preparing an accessible Word document

The key to creating an accessible PDF is starting with an accessible Word document. Most accessibility features in Word carry over to the PDF when you export correctly.

Use heading styles

Headings provide structure that screen readers use for navigation. Always use Word's built-in heading styles rather than just making text bold or large.

  • Use Heading 1 for main titles
  • Use Heading 2 for major sections
  • Use Heading 3, 4, etc. for subsections
  • Don't skip heading levels (e.g., don't jump from Heading 1 to Heading 3)

How to apply: Select text, then choose a heading style from the Styles group on the Home tab.

Use real lists

Create lists using Word's bullet or numbering tools, not by manually typing bullets or numbers.

  • Bulleted lists: Use for items in no particular order
  • Numbered lists: Use for steps or items in a sequence

How to apply: Select your text, then click the Bullets or Numbering button on the Home tab.

Add alternative text to images

Images need text descriptions so screen reader users know what they show.

How to add alt text:

  1. Right-click the image and select "View Alt Text" (or "Edit Alt Text")
  2. Type a brief but meaningful description in the Alt Text pane
  3. If the image is purely decorative, check "Mark as decorative"

What makes good alt text:

  • Describe what's important about the image in context
  • Keep it concise (usually 1-2 sentences)
  • Don't start with "Image of..." or "Picture of..."
  • For charts/graphs, describe the key data or trend

Use descriptive link text

Screen readers can list all links on a page. Each link should make sense out of context.

  • Good: "Download the accessibility guide" or "Read the full report"
  • Bad: "Click here" or "Read more" or using the URL itself

How to create good links:

  1. Select meaningful text that describes the link destination
  2. Press Ctrl+K (or right-click and choose "Link")
  3. Enter or paste the URL

Use Word's table features

If you need tables, use Word's Insert Table feature and mark the header row:

  1. Insert a table using Insert > Table
  2. Select the first row
  3. Check "Repeat as header row" on the Table Design tab

Using the Accessibility Checker

Word's built-in Accessibility Checker identifies common accessibility issues before you export to PDF.

How to run the Accessibility Checker

  1. Click the Review tab on the ribbon
  2. Click Check Accessibility
  3. The Accessibility pane opens on the right side

Understanding the results

The checker groups issues into three categories:

  • Errors: Content that is very difficult or impossible for people with disabilities to access (e.g., missing alt text)
  • Warnings: Content that may be difficult to access (e.g., unclear link text)
  • Tips: Suggestions that improve accessibility (e.g., using simple table structure)

Fixing issues

Click on any issue in the list to:

  • Jump to that location in your document
  • See why it's an issue
  • Learn how to fix it

Tip: Fix errors first, then warnings, then review tips as time allows.

Keep the Accessibility Checker open: Check "Keep accessibility checker running while I work" at the bottom of the pane. This provides real-time feedback as you create content.

Setting document properties

Document properties help assistive technology users understand and navigate your document. They also improve searchability.

How to set properties

  1. Click File > Info
  2. On the right side, you'll see Properties
  3. Fill in at least:
    • Title: The document's main title (will appear in the PDF)
    • Author: Who created the document
    • Subject: Brief description of the document's purpose
  4. Click "Show All Properties" to see additional fields

Why properties matter

  • Screen readers announce the document title when opening the PDF
  • Properties appear in PDF readers' document information
  • They help users verify they have the right document
  • Search engines and document management systems use this metadata

Exporting to PDF with accessibility tags

Using the correct export method is crucial for creating accessible PDFs from Word.

Step-by-step export process

  1. Click File > Save As
  2. Choose where to save the file
  3. In the "Save as type" dropdown, select PDF (*.pdf)
  4. Important: Click the Options button
  5. In the Options dialog, make sure these are checked:
    • "Document structure tags for accessibility" - This is the most critical setting
    • "ISO 19005-1 compliant (PDF/A)" if archival format is needed
  6. Click OK
  7. Click Save

Critical: Always use "Save As" with Options

Do not use:

  • "Export" to PDF – This may not include all accessibility features
  • "Print to PDF" – This creates an image-based PDF without any accessibility features
  • Third-party PDF converters – These often strip accessibility information

Verifying your PDF

After exporting, quickly check that accessibility carried over:

  1. Open the PDF in Adobe Acrobat Reader (free version is fine)
  2. Try using the keyboard:
    • Press Tab to navigate between links and form fields
    • Use arrow keys to read text
  3. In Acrobat Reader, check File > Properties > Description to verify your title and metadata transferred

Essential tips for accessible Word PDFs

Do's

  • Do use built-in Word features – heading styles, lists, tables, etc.
  • Do run the Accessibility Checker before exporting
  • Do set document properties (title, author, subject)
  • Do add alt text to all informative images
  • Do use descriptive link text
  • Do check "Document structure tags" when exporting
  • Do test the PDF after exporting, at least briefly

Don'ts

  • Don't use manual formatting instead of styles (e.g., bold text instead of headings)
  • Don't skip heading levels in your document structure
  • Don't use "Print to PDF" – it strips all accessibility features
  • Don't leave images without alt text (unless they're truly decorative)
  • Don't use color alone to convey meaning
  • Don't use vague link text like "click here"
  • Don't create fake lists using manual bullets or numbers

Quick checklist before exporting

  • ✓ Document uses heading styles properly
  • ✓ All images have alt text or are marked decorative
  • ✓ Links have descriptive text
  • ✓ Accessibility Checker shows no errors
  • ✓ Document properties are set
  • ✓ Lists and tables use Word's built-in features

Remember

An accessible Word document + correct export settings = an accessible PDF. Take time to build accessibility into your Word document, and the PDF export will preserve it. Fixing accessibility issues in PDF later is much harder than creating an accessible Word document from the start.
