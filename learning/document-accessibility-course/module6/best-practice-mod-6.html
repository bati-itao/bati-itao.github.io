PDF structure and tagging

  • Create PDFs from well-structured source documents (Word, InDesign, etc.) rather than from scanned images
  • Use proper heading styles in source documents to ensure logical heading hierarchy in PDFs
  • Enable accessibility options when exporting to PDF from source applications
  • Ensure all content is properly tagged with appropriate PDF tags (P for paragraphs, H1-H6 for headings, L for lists, etc.)
  • Use auto-tagging as a starting point, but manually review and correct the tag structure
  • Create a logical tag tree that reflects the document's hierarchical structure
  • Remove any untagged content or mark decorative elements as artifacts

Reading order

  • Ensure reading order follows a logical sequence that makes sense without visual context
  • Test reading order using screen readers or Adobe Acrobat's Read Out Loud feature
  • Check that multi-column layouts read down each column before moving to the next
  • Verify that sidebar content appears in appropriate context, not disrupting main content flow
  • Ensure headers and footers don't interrupt main content reading sequence
  • Use Adobe Acrobat's Content panel to reorder content when necessary
  • Place table content in logical reading order (left to right, top to bottom)

Alternative text

  • Provide descriptive alternative text for all informative images, charts, and graphics
  • Add alternative text in source applications when possible, rather than in the PDF
  • Keep alternative text concise but complete, focusing on essential information
  • Describe the purpose and meaning of images, not just their appearance
  • For complex images like charts, provide detailed descriptions in addition to alt text
  • Mark decorative images as artifacts so they're ignored by assistive technologies
  • Ensure images of text include the actual text content in the alternative text

Accessibility checking

  • Run Adobe Acrobat's Accessibility Checker to identify common issues
  • Address all failed checks and review warnings for potential problems
  • Test documents with screen readers (NVDA, JAWS) to verify user experience
  • Check color contrast meets WCAG 2.1 AA requirements (4.5:1 for normal text)
  • Verify that all interactive elements are keyboard accessible
  • Test navigation features like heading jumps and bookmark functionality
  • Use additional tools like PAC (PDF Accessibility Checker) for comprehensive analysis
  • Validate that form fields have proper labels and descriptions

Remediation

  • Start with the highest priority issues identified by accessibility checkers
  • Fix structural problems (missing tags, incorrect reading order) before minor issues
  • Add missing alternative text for images and graphics
  • Correct table structure and ensure proper header relationships
  • Improve form field labels and descriptions for better usability
  • Set appropriate tab order for interactive elements
  • Add bookmarks for long documents to improve navigation
  • Test fixes with assistive technologies to verify effectiveness
  • Document remediation process for future reference and consistency
