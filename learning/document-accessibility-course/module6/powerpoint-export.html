Ensuring slide deck accessibility

PowerPoint presentations converted to PDF are commonly used for sharing presentations after meetings, for reports, or as handouts. Before exporting to PDF, your presentation must be accessible in PowerPoint first.

📚 Prerequisites: PowerPoint accessibility fundamentals

This lesson assumes you've already made your PowerPoint presentation accessible. If you need to learn or review PowerPoint accessibility basics, see:

  • Slide Master & Templates - Foundation setup and accessible templates
  • Slide Layouts & Structure - Using built-in layouts and proper structure
  • Reading Order & Navigation - Ensuring logical content flow
  • Alt Text & Long Descriptions - Making images accessible
  • Color and Contrast - Meeting contrast requirements

Quick pre-export accessibility verification

Before exporting to PDF, verify these essential elements are in place:

  • Slide titles: Every slide has a unique, descriptive title (see Slide Layouts & Structure)
  • Alt text: All images have appropriate descriptions (see Alt Text & Long Descriptions)
  • Color contrast: Text meets minimum contrast ratios (see Color and Contrast)
  • Reading order: Content flows logically (see Reading Order & Navigation)
  • Built-in layouts: Using PowerPoint's structured layouts (see Slide Layouts & Structure)

Checking reading order before export

Reading order determines how screen readers navigate through your PDF content. Issues with reading order in PowerPoint will carry over to the exported PDF.

📖 Learn about reading order

For detailed guidance on checking and fixing reading order in PowerPoint, see Reading Order & Navigation in Module 3.

Quick reading order check for PDF export

Before exporting, do a final reading order verification:

  1. Open the Selection Pane (Home tab > Arrange > Selection Pane)
  2. Check that objects are listed in logical reading order (bottom to top in the list)
  3. Use Tab key navigation to verify the sequence makes sense
  4. Pay special attention to slides with multiple text boxes or complex layouts

Common reading order issues that affect PDFs

  • Text boxes read before titles: Can confuse PDF navigation
  • Multi-column content: May not flow logically in PDF
  • Floating images with captions: Captions may be separated from images
  • Sidebar content: May interrupt main content flow

Final Accessibility Checker run

Before exporting to PDF, run PowerPoint's Accessibility Checker one final time to catch any remaining issues.

Quick Accessibility Checker workflow

  1. Click the Review tab > Check Accessibility
  2. Address any Errors (these will block PDF accessibility)
  3. Review Warnings and fix when possible
  4. Consider Tips for improved accessibility

🔍 Detailed Accessibility Checker guidance

For comprehensive information about using the Accessibility Checker in PowerPoint, including how to interpret and fix specific issues, see the Accessibility Checker sections in:

  • Slide Master & Templates
  • Global M365 Document Techniques

Critical issues to resolve before PDF export

  • Missing slide titles - PDFs need titles for navigation
  • Missing alt text - Images will be inaccessible in PDF
  • Poor color contrast - Text may be unreadable
  • Reading order warnings - Will affect PDF navigation

Exporting to PDF with accessibility tags

PowerPoint offers multiple ways to create PDFs. The method you choose makes a significant difference in accessibility.

Recommended method: Save As PDF

This is the most reliable method for accessible PDFs from PowerPoint.

  1. Click File > Save As
  2. Choose where to save your file
  3. In the "Save as type" dropdown, select PDF (*.pdf)
  4. Important: Click the Options button
  5. In the Options dialog, verify these settings:
    • "Document structure tags for accessibility" is checked (this is critical)
    • Choose what to publish: "All" slides or a range
    • Choose whether to include hidden slides
  6. Click OK
  7. Click Save

Alternative method: Create PDF/XPS

This method also works but has fewer options:

  1. Click File > Export
  2. Click Create PDF/XPS Document
  3. Click Create PDF/XPS
  4. Click Options and ensure "Document structure tags for accessibility" is checked
  5. Click OK, then Publish

Critical: Avoid "Print to PDF"

Never use "Print to PDF" – This creates an image-based PDF without any accessibility features. The resulting PDF will be:

  • Completely inaccessible to screen readers
  • Missing all tags and structure
  • Unable to be navigated by keyboard
  • Non-compliant with accessibility requirements

Always use Save As or Create PDF/XPS with tags enabled.

Setting PDF options

In the Options dialog, consider these settings:

  • Slides: Choose which slides to include
  • Publish what: "Slides" (not "Handouts" or "Notes Pages" unless specifically needed)
  • Frame slides: Generally leave unchecked for cleaner output
  • Include hidden slides: Only if needed

After exporting

Open your PDF and do a quick check:

  1. Open in Adobe Acrobat Reader (free version)
  2. Try keyboard navigation – press Tab to move between content
  3. Check File > Properties to see if document title is set
  4. If available, use the Accessibility Checker in Acrobat Pro

Essential tips for accessible PowerPoint PDFs

PDF export-specific considerations

  • Animations don't carry over to PDF: Ensure content makes sense without animations
  • Transitions are ignored: PDF shows static slides only
  • Speaker notes aren't included: Put essential information on slides or in a separate document
  • Links work in PDFs: Hyperlinks on slides remain functional
  • Document properties transfer: PDF title comes from presentation title

Quick pre-export checklist

Before exporting to PDF, verify:

  • ✓ PowerPoint accessibility fundamentals complete (see Module 3)
  • ✓ Accessibility Checker shows no critical errors
  • ✓ Reading order verified on complex slides
  • ✓ Content is clear without animations
  • ✓ Document title set in File > Info > Properties

Key takeaway

PDF accessibility starts in PowerPoint. Focus on proper slide structure, meaningful titles, alt text, and reading order during PowerPoint creation. Then use the correct export method ("Save As PDF" with document structure tags) to preserve that accessibility in the final PDF.
