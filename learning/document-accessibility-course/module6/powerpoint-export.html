Ensuring slide deck accessibility

PowerPoint presentations converted to PDF are commonly used for sharing presentations after meetings, for reports, or as handouts. Making your slides accessible before exporting ensures the PDF will work for everyone.

Use meaningful slide titles

Every slide must have a unique, descriptive title. Screen readers use slide titles to help users navigate the document.

  • Good titles: "Q1 Revenue Results", "Project Timeline", "Contact Information"
  • Bad titles: "Slide 1", "Untitled Slide", or identical titles on multiple slides

How to add/edit titles:

  1. Click the title placeholder on the slide
  2. Type a descriptive title
  3. If you don't want the title to appear visually, use the Selection Pane to hide it while keeping it for screen readers

Add alternative text to images

All images, charts, SmartArt, and other visual elements need text descriptions.

How to add alt text:

  1. Right-click the image and select "View Alt Text" (or "Edit Alt Text")
  2. Type a meaningful description in the Alt Text pane
  3. For decorative images, check "Mark as decorative"

What to describe:

  • Photos: Describe who/what is shown and why it's relevant
  • Charts: Summarize the key data point or trend
  • Diagrams: Explain the relationship or process shown
  • Logos: Usually just the organization name is sufficient

Use sufficient color contrast

Text must have enough contrast against its background for people with low vision or color blindness.

  • Minimum contrast ratio: 4.5:1 for normal text, 3:1 for large text (18pt or larger)
  • Avoid: Light text on light backgrounds, or dark text on dark backgrounds
  • Don't rely on color alone: Use text labels, patterns, or icons in addition to color

Check contrast: The Accessibility Checker will flag some contrast issues, but manually review your slides too.

Keep slides simple and uncluttered

Complex or cluttered slides are harder for everyone to understand, especially people with cognitive disabilities.

  • Limit content per slide: One main idea per slide
  • Use simple layouts: Avoid overlapping text boxes
  • Remove decorative elements: Only include images that add meaning
  • Use built-in layouts: PowerPoint's layouts ensure proper reading order

Use built-in layouts and placeholders

PowerPoint's slide layouts have proper reading order built in. When you add your own text boxes, the reading order may become confusing.

  • Use: Title placeholder, content placeholder, built-in layouts
  • Avoid: Free-floating text boxes when possible
  • If you must use text boxes: Check and fix reading order (see next section)

Checking reading order

Reading order is the sequence in which a screen reader presents slide content. If the order is wrong, content may be confusing or nonsensical.

Why reading order matters

Screen readers read content in the order it appears in the Selection Pane, not based on visual position. A slide might look organized visually but be read in a jumbled order.

How to check and fix reading order

  1. On the Home tab, click Arrange > Selection Pane
  2. The Selection Pane shows all objects on the slide
  3. Screen readers read from bottom to top in this list
  4. To reorder:
    • Drag objects up or down in the list, or
    • Select an object and use the up/down arrows
  5. The typical reading order should be:
    • Slide title (top in Selection Pane = read last)
    • Content in logical order
    • Any additional text or images

When to check reading order

Check reading order when slides have:

  • Multiple text boxes
  • Images with captions
  • Multi-column layouts
  • Custom layouts or designs

Using the Accessibility Checker

PowerPoint's Accessibility Checker identifies common issues before you export to PDF.

How to run the Accessibility Checker

  1. Click the Review tab on the ribbon
  2. Click Check Accessibility
  3. The Accessibility pane opens on the right

Alternatively: In recent versions, you may see an Accessibility icon in the status bar at the bottom of the window. Click it to open the checker.

Understanding the results

Issues are grouped as:

  • Errors: Must fix – content that blocks access (e.g., missing slide titles, images without alt text)
  • Warnings: Should fix – content that may be difficult to access (e.g., low contrast, unclear table headers)
  • Tips: Consider fixing – suggestions for better accessibility

Fix issues slide by slide

Click on any issue to:

  • Jump to that slide
  • See why it's flagged
  • Get instructions on how to fix it

Keep the checker running: Check "Keep accessibility checker running while I work" at the bottom of the pane. This gives you real-time feedback as you build your presentation.

Common issues the checker finds

  • Missing slide titles
  • Missing alt text on images
  • Poor color contrast
  • Unclear reading order
  • Tables without header rows
  • Hard-to-read fonts or font sizes

Exporting to PDF with accessibility tags

PowerPoint offers multiple ways to create PDFs. The method you choose makes a significant difference in accessibility.

Recommended method: Save As PDF

This is the most reliable method for accessible PDFs from PowerPoint.

  1. Click File > Save As
  2. Choose where to save your file
  3. In the "Save as type" dropdown, select PDF (*.pdf)
  4. Important: Click the Options button
  5. In the Options dialog, verify these settings:
    • "Document structure tags for accessibility" is checked (this is critical)
    • Choose what to publish: "All" slides or a range
    • Choose whether to include hidden slides
  6. Click OK
  7. Click Save

Alternative method: Create PDF/XPS

This method also works but has fewer options:

  1. Click File > Export
  2. Click Create PDF/XPS Document
  3. Click Create PDF/XPS
  4. Click Options and ensure "Document structure tags for accessibility" is checked
  5. Click OK, then Publish

Critical: Avoid "Print to PDF"

Never use "Print to PDF" – This creates an image-based PDF without any accessibility features. The resulting PDF will be:

  • Completely inaccessible to screen readers
  • Missing all tags and structure
  • Unable to be navigated by keyboard
  • Non-compliant with accessibility requirements

Always use Save As or Create PDF/XPS with tags enabled.

Setting PDF options

In the Options dialog, consider these settings:

  • Slides: Choose which slides to include
  • Publish what: "Slides" (not "Handouts" or "Notes Pages" unless specifically needed)
  • Frame slides: Generally leave unchecked for cleaner output
  • Include hidden slides: Only if needed

After exporting

Open your PDF and do a quick check:

  1. Open in Adobe Acrobat Reader (free version)
  2. Try keyboard navigation – press Tab to move between content
  3. Check File > Properties to see if document title is set
  4. If available, use the Accessibility Checker in Acrobat Pro

Essential tips for accessible PowerPoint PDFs

Do's

  • Do give every slide a unique, descriptive title
  • Do add alt text to all images, charts, and SmartArt
  • Do use built-in slide layouts instead of custom text boxes
  • Do check color contrast – ensure text is readable
  • Do run the Accessibility Checker before exporting
  • Do check reading order on complex slides
  • Do use "Save As" with document structure tags
  • Do keep slides simple and uncluttered

Don'ts

  • Don't use "Print to PDF" – it strips all accessibility features
  • Don't leave slides without titles or use generic titles like "Slide 1"
  • Don't forget alt text on images (unless purely decorative)
  • Don't rely only on color to convey information
  • Don't use low contrast color combinations
  • Don't create cluttered, complex slides with overlapping elements
  • Don't skip checking reading order on slides with multiple objects

PowerPoint-specific considerations

  • Animations don't carry over to PDF: Ensure content makes sense without animations
  • Transitions are ignored: PDF shows static slides only
  • Speaker notes aren't included: Put essential information on slides or in a separate document
  • Links work in PDFs: Hyperlinks on slides remain functional

Quick pre-export checklist

  • ✓ Every slide has a unique, descriptive title
  • ✓ All images have appropriate alt text
  • ✓ Color contrast is sufficient
  • ✓ Reading order is logical
  • ✓ Accessibility Checker shows no errors
  • ✓ Content is clear without animations
  • ✓ Slides use built-in layouts where possible

Key takeaway

A well-structured, accessible PowerPoint presentation + correct export method = an accessible PDF. Focus on slide titles, alt text, reading order, and contrast. Then use "Save As PDF" with document structure tags enabled. Avoid "Print to PDF" – always use the proper export methods described above.
