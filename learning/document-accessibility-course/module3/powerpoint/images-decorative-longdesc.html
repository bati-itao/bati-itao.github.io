Scope note

Note: The global M365 section covers general guidance on alternative text. Here we focus on PowerPoint-specific workflows for decorative and complex visuals that require special handling in presentation contexts.

This section builds upon the foundation concepts covered in Module 2: Images and Alt Text.

PowerPoint presentations often include complex visual elements that require specialized accessibility approaches beyond basic alt text. This includes decorative graphics that should be hidden from screen readers, complex diagrams that need detailed descriptions, and interactive visual elements that must be properly integrated into the slide's reading order.

Decorative images (Windows)

Decorative images serve aesthetic purposes but don't convey essential information. Properly marking them as decorative prevents screen reader clutter and allows users to focus on meaningful content.

How to mark images as decorative

  1. Select the image → right-click → View Alt Text
  2. Check 'Mark as decorative' for purely ornamental graphics
  3. When marked as decorative, the alt text field becomes unavailable (grayed out)
  4. Screen readers will skip these images entirely

What qualifies as decorative in PowerPoint?

✅ Typically decorative:

  • Brand logos (unless they're the main subject)
  • Background patterns and textures
  • Design flourishes and ornamental shapes
  • Repeating visual elements used for layout
  • Icons that duplicate text (bullet point icons alongside text)

❌ Usually not decorative:

  • Photos showing process steps
  • Diagrams explaining concepts
  • Charts and graphs with data
  • Screenshots demonstrating software
  • Icons conveying unique information

Reading order management for decorative images

  1. Ensure decorative items are placed later in reading order (or hidden) so they don't interrupt focus
  2. Use the Selection Pane to move decorative elements to the top of the object list (read last)
  3. Consider hiding decorative elements using the eye icon in Selection Pane during content review
  4. Group decorative elements together if they're related

⚠️ Common mistake: Over-marking as decorative

Avoid marking images as decorative when they:

  • Provide context or emotional tone important to understanding
  • Show examples or demonstrations
  • Contain text or data (even if also described elsewhere)
  • Serve as navigation elements or interactive components

Complex visuals (diagrams, dense charts)

Complex visuals require a multi-layered accessibility approach: concise alt text for identification, detailed descriptions for full understanding, and alternative formats for data access.

Step 1: Provide concise alt text

  1. Name the visual and its purpose (e.g., 'Timeline of milestones', 'Organizational chart', 'Process flow diagram')
  2. Include the main takeaway or conclusion if space allows
  3. Keep to 125 characters or less for the primary alt text
  4. Don't start with "Image of" or "Picture showing" - be direct

Step 2: Provide detailed descriptions

For complex visuals, use multiple strategies to provide full access:

Option A: Speaker notes

  • Put longer explanations or data summaries in the speaker notes
  • Include step-by-step descriptions of processes
  • List all data points for charts and graphs
  • Describe spatial relationships in diagrams

Option B: Following slide

  • Create a dedicated slide with text description of the visual
  • Break complex diagrams into step-by-step explanations
  • Use bullet points to describe process flows
  • Include this slide in your presentation flow

Option C: On-slide text summary

  • Include a brief takeaway directly on the slide
  • Highlight the most important data points
  • Use callout boxes for key insights
  • Position text logically in reading order

Step 3: Alternative data formats

  1. When possible, present the underlying data in a simple table on the slide or linked Excel sheet
  2. For processes, create a numbered list of steps
  3. For organizational charts, provide a hierarchical text list
  4. For timelines, create a chronological list with dates and events

Examples of complex visual descriptions

Example 1: Process diagram

Alt text: "Customer service workflow diagram showing 5-step resolution process"

Detailed description (speaker notes):

  1. Customer contacts support via phone, email, or chat
  2. Support agent logs issue in ticketing system
  3. Issue is categorized and routed to appropriate team
  4. Team member investigates and implements solution
  5. Customer receives resolution notification and feedback request

Example 2: Complex chart

Alt text: "Revenue growth chart showing 15% increase over 4 quarters"

On-slide summary: "Key insight: Revenue grew consistently each quarter, with strongest growth in Q4 (22%)"

Data table: Provide exact figures in speaker notes or linked spreadsheet

Icons and symbols

Functional vs. decorative icons

The same icon can be functional or decorative depending on context:

Functional icons (need alt text):

  • Navigation elements (home, back, forward)
  • Status indicators (warning, error, success)
  • Action triggers (print, save, share)
  • Category identifiers (when text doesn't specify)

Decorative icons (mark as decorative):

  • Bullet point decorations (when text is clear)
  • Visual reinforcement of clear text labels
  • Aesthetic elements in headers/footers
  • Brand symbols used for layout

Best practices for functional icons

  • Describe the function, not the appearance: "Delete item" not "Trash can icon"
  • Use standard terminology: Consistent with other applications
  • Include state information: "Expand menu (currently collapsed)"
  • Consider cultural differences: Symbols may have different meanings in different regions

Symbol accessibility considerations

  • Text alternatives for complex symbols: Mathematical notation, chemical formulas, etc.
  • Cultural symbol interpretation: Explain symbols that may not be universally understood
  • Color-dependent symbols: Ensure meaning isn't lost without color
  • Size and contrast: Ensure symbols are visible and distinguishable

Reading order considerations

Logical placement of images in content flow

Images should be positioned in the reading order where they make most sense contextually:

For explanatory images:

  1. Place images immediately after the text they illustrate
  2. If the image is referenced in text ("as shown in the image below"), ensure the reading order matches
  3. Group related images and captions together

For process illustrations:

  1. Position process diagrams before step-by-step explanations
  2. Or place them after the explanation as visual reinforcement
  3. Be consistent within your presentation

Managing image-heavy slides

  • Prioritize essential images: Consider if all images are necessary
  • Group related visuals: Use PowerPoint's grouping feature for related elements
  • Create logical sections: Organize images into meaningful clusters
  • Use descriptive names: Name image objects clearly in the Selection Pane

Speaker notes strategy

When to use speaker notes for image descriptions

  • Complex technical diagrams that require detailed explanation
  • Data visualizations with multiple data points
  • Artistic or conceptual images that need interpretation
  • Screenshots with multiple interface elements

Structure for speaker notes descriptions

  1. Overview: General description of what the image shows
  2. Key elements: Important components and their relationships
  3. Data/details: Specific information, measurements, or data points
  4. Context/significance: Why this image matters to your presentation

Making speaker notes accessible

  • Use clear, structured writing: Short sentences and logical flow
  • Include all essential information: Don't assume visual elements are self-explanatory
  • Provide data alternatives: Tables, lists, or linked resources
  • Test with screen readers: Verify that notes are accessible when needed

Quick checklist

Before finalizing slides with images:

  • ☐ Decorative images marked as decorative
  • ☐ Complex visuals have a meaningful label and a longer description elsewhere if needed
  • ☐ Functional icons have descriptive alt text
  • ☐ Images are positioned logically in reading order
  • ☐ Speaker notes include detailed descriptions for complex visuals
  • ☐ Alternative data formats provided where appropriate
  • ☐ All images meet color contrast requirements
  • ☐ Image object names are descriptive in Selection Pane
