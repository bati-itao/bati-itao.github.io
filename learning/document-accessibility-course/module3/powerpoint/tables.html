Why it matters

Tables can be hard to navigate in slides. Keep them simple and readable. Complex tables that work well in documents may become confusing or inaccessible when presented in PowerPoint format.

Keep tables simple

  1. Use Insert → Table (avoid images of tables)
  2. Limit to simple grids; avoid merged/split cells
  3. Use the 'Header Row' table style option so headers are visually distinct (note: semantic headers in PPT are limited)
  4. Provide the table data in notes or a linked Excel file for more accessible navigation

Table design principles

  • Clear headers: Use the first row for column headers
  • Logical structure: Organize data in a predictable pattern
  • Appropriate size: Ensure text remains readable when projected
  • High contrast: Use sufficient contrast between text and background

Creating accessible tables

Step-by-step process:

  1. Insert table using Insert → Table
  2. Add clear, descriptive column headers
  3. Apply Header Row style from Table Design
  4. Ensure sufficient font size (minimum 18pt)
  5. Test readability and contrast
  6. Provide alternative formats if complex

Formatting considerations:

  • Font size: Larger than body text for readability
  • Cell padding: Adequate white space around content
  • Alignment: Consistent alignment within columns
  • Borders: Clear distinction between cells

Alternative formats

When tables are complex:

  • Break into multiple slides: Split large tables across several slides
  • Summary format: Present key findings rather than all data
  • Linked Excel file: Provide full data in accessible spreadsheet
  • Speaker notes: Include table data in notes for screen reader access

Visual alternatives:

  • Charts: Convert data to charts when appropriate
  • Infographics: Visual representation of key data points
  • Lists: Convert simple tables to bulleted lists
  • Narrative text: Describe data trends in paragraph form

Quick checklist

Before finalizing tables:

  • ☐ No merged cells
  • ☐ Sufficient text size and contrast in cells
  • ☐ Data available in an alternative format if complex
  • ☐ Clear column headers using Header Row style
  • ☐ Logical data organization
  • ☐ Readable when projected or viewed on small screens
