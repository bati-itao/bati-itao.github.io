What it is

Slide layouts and structure are about organizing your content so screen readers and other assistive technology can understand it. Good structure means using PowerPoint's built-in layouts instead of blank slides, putting content in the right placeholders, and creating titles that make sense.

When you use proper structure, people using screen readers can navigate your slides easily, skip to the content they want, and understand how information relates to each other.

Why structure matters

Proper slide structure follows government accessibility requirements and provides these benefits:

  • Screen reader navigation: Users can jump between slides and find content quickly
  • Logical reading order: Content is read in the right sequence
  • Clear relationships: Titles, headings, and content connect properly
  • Keyboard navigation: Tab key moves through content in logical order
  • Better for everyone: Organized content is easier for all users to follow

Avoid these structure problems

  • Using blank slides (no built-in structure)
  • Adding text boxes instead of using placeholders
  • Slides without titles
  • Content that doesn't follow logical order

How to use placeholders

Placeholders are pre-made areas for different types of content. Here's how to use them correctly:

  1. Always use the title placeholder:
    • Click in the title area at the top of each slide
    • Add a unique, descriptive title
    • Never leave title placeholders empty
    • Don't delete title placeholders
  2. Use content placeholders for all body content:
    • Click in the content area to add text, images, or other content
    • Don't add separate text boxes - use the placeholder instead
    • Content placeholders handle multiple types: text, images, charts

Slide Layouts and Structure checklist

  • ☐ Use built-in slide layouts (not blank slides)
  • ☐ Every slide has a unique, descriptive title
  • ☐ Content uses placeholder boxes from layouts
  • ☐ Visual hierarchy is clear and consistent
  • ☐ All text content is available in Outline View
  • ☐ Tab navigation follows logical sequence
