Using built-in slide layouts

Built-in slide layouts are the foundation of accessible PowerPoint presentations. They provide proper structure that assistive technologies can understand and navigate efficiently.

Why built-in layouts matter

  • Semantic structure: Layouts define clear relationships between content elements
  • Predictable navigation: Screen readers can reliably find titles, content, and other elements
  • Responsive design: Content adapts better to different screen sizes and zoom levels
  • Consistency: Uniform structure across slides helps all users follow along

❌ Avoid blank slides

Never use the "Blank" slide layout unless absolutely necessary. Blank slides require manual positioning of all elements, which often results in accessibility problems:

  • No defined title area
  • Unpredictable reading order
  • Difficult navigation for assistive technologies
  • Content may not be properly identified

Choosing the right layout

PowerPoint 365 offers several built-in layouts designed for different content types:

Title Slide
Perfect for presentation opening, section dividers, or major topic introductions
Title and Content
Most versatile layout - use for text, images, charts, or mixed content
Section Header
Ideal for dividing presentation into major sections
Two Content
Great for comparisons, before/after, or side-by-side information
Comparison
Specialized layout for comparing two concepts or sets of information
Content with Caption
Perfect when you need a large image with explanatory text

Applying layouts in PowerPoint 365

  1. Select the slide you want to modify
  2. Go to Home > Layout (in the Slides group)
  3. Choose the layout that best matches your content
  4. PowerPoint will automatically map existing content to appropriate placeholders

✅ Pro tip: Layout first, design second

Always choose your layout based on content structure, not visual appearance. You can modify colors, fonts, and spacing later while maintaining the accessible structure.

Working with placeholders

Placeholders are predefined areas in slide layouts that tell PowerPoint (and assistive technologies) what type of content belongs where.

Types of placeholders

  • Title placeholder: Defines the slide's main heading
  • Content placeholder: For body text, lists, images, charts, or other media
  • Text placeholder: Specifically for text content
  • Picture placeholder: Optimized for images
  • Chart placeholder: For data visualizations
  • Table placeholder: For tabular data

Best practices for placeholders

Use placeholders correctly

  • Title placeholders: Always use for slide titles, never leave empty
  • Content placeholders: Use for all body content instead of inserting independent text boxes
  • Placeholder purpose: Use placeholders for their intended content type when possible

When you need to add content outside placeholders

Sometimes you need additional content areas. Here's how to do it accessibly:

  1. Try a different layout first: Another layout might better fit your needs
  2. Modify Slide Master: Add permanent placeholders for reusable content
  3. Use text boxes sparingly: If you must add text boxes, ensure they're in logical reading order
  4. Group related elements: Use PowerPoint's grouping feature for complex layouts

⚠️ Common placeholder mistakes

  • Leaving title placeholders empty: Every slide needs a title
  • Using text boxes instead of placeholders: Breaks the semantic structure
  • Deleting unused placeholders: Just leave them empty or choose a different layout
  • Overlapping content areas: Causes confusion for screen readers

Working with content placeholders

Content placeholders are your most flexible tool. They can contain:

Text content

  • Bulleted lists
  • Numbered lists
  • Body paragraphs
  • Formatted text

Media content

  • Images and photos
  • Charts and graphs
  • SmartArt diagrams
  • Videos and audio

When you click on a content placeholder, PowerPoint 365 shows options for different content types. This guided approach helps maintain accessibility structure.

Creating meaningful slide titles

Slide titles are crucial for navigation and accessibility. They serve as headings that screen reader users can jump between, making your presentation much easier to navigate.

Why every slide needs a title

  • Navigation landmarks: Screen readers use titles to navigate between slides
  • Context understanding: Titles help users understand where they are in the presentation
  • Search functionality: Titles are indexed for searching within the presentation
  • Outline view: Titles create a meaningful presentation outline

Writing effective slide titles

Title writing principles

  • Be descriptive: Titles should clearly indicate the slide's main topic
  • Be unique: Each slide should have a different title (unless part of a series)
  • Be concise: Aim for 5-10 words when possible
  • Use parallel structure: Maintain consistent title format throughout sections

Title examples

❌ Poor titles
  • Introduction
  • Data
  • Chart
  • Next Steps
  • Questions
✅ Good titles
  • Welcome to Customer Service Excellence
  • Q3 Sales Performance by Region
  • Revenue Growth: 2020-2024
  • Implementation Timeline and Milestones
  • Q&A and Contact Information

Hidden titles for visual design

Sometimes visual design requires that titles not be visible on the slide, but you still need them for accessibility:

  1. Add your descriptive title as normal
  2. Select the title text
  3. Go to Home > Font > Text Effects > Outline > No Outline
  4. Set the text color to match the background
  5. Or move the title placeholder off the slide (but keep it accessible)

💡 Alternative: Use subtitle areas

If your design needs visual impact without visible titles, consider:

  • Using a large, descriptive subtitle that serves as the main heading
  • Creating a visual hierarchy with size and color rather than hiding text
  • Adding the title as the first line of your content area

Title consistency across presentations

For multi-part presentations or series:

  • Use consistent prefixes: "Module 1:", "Session 2:", etc.
  • Maintain parallel structure: If one slide is "Creating Tables", another should be "Creating Charts"
  • Include context: Help users understand where they are in the larger presentation

Content structure and hierarchy

Proper content hierarchy helps all users understand the relationship between different pieces of information on your slides.

Using headings effectively

  • Slide title = H1: The slide title serves as the main heading
  • Major sections = H2: Use for main content sections within a slide
  • Subsections = H3: For sub-topics under major sections
  • Avoid deep nesting: Rarely go beyond H3 on a single slide

Creating headings in PowerPoint 365

  1. Select the text you want to make a heading
  2. Go to Home > Styles
  3. Choose the appropriate heading level
  4. Or use the outline view to organize content hierarchically

Visual hierarchy best practices

  • Size relationships: Larger text for more important headings
  • Color consistency: Use consistent colors for similar heading levels
  • White space: More space around higher-level headings
  • Font weight: Bold for headings, regular for body text

✅ Testing your hierarchy

To test if your hierarchy is clear:

  1. Switch to View > Outline View
  2. Look at the text-only outline - does it make sense?
  3. Use Tab key to navigate through the slide - is the order logical?
  4. Ask someone else to identify the main points from the visual hierarchy

Common accessible layout patterns

Here are some layout patterns that work well for accessibility while maintaining visual appeal.

Text-heavy slides

  • Use "Title and Content" layout
  • Break up text with:
    • Bulleted or numbered lists
    • Short paragraphs (3-4 sentences max)
    • Subheadings to organize topics
    • White space for visual breathing room

Image-focused slides

  • Use "Content with Caption" layout
  • Always include:
    • Descriptive alt text for the image
    • Caption text that explains the image's relevance
    • Context about why the image is important

Data presentation slides

  • Use "Title and Content" with chart placeholder
  • Include:
    • Clear chart title describing the data
    • Alt text summarizing key findings
    • Data table as backup (can be hidden but available)
    • Explanation of trends or important points

Comparison slides

  • Use "Comparison" or "Two Content" layout
  • Structure with:
    • Clear headings for each side of the comparison
    • Parallel structure in content organization
    • Consistent formatting on both sides
    • Summary of key differences or similarities

Troubleshooting layout issues

Common problems and solutions

Content doesn't fit in placeholders

  • Try a different layout: Two Content might work better than Title and Content
  • Reduce content: Consider splitting into multiple slides
  • Adjust font size: But ensure it remains readable (18pt minimum)
  • Use outline format: Bulleted lists take up less space than paragraphs

Reading order seems wrong

  • Check with Tab key: Navigate through the slide to see the order
  • Use Reading Order Pane: View > Reading Order Pane to see and adjust order
  • Simplify the layout: Complex layouts often have order issues
  • Group related elements: Use PowerPoint's grouping feature

Accessibility Checker flags layout issues

  • Missing slide title: Always add a descriptive title
  • Reading order problems: Use the Reading Order Pane to fix
  • Text outside placeholders: Move content into appropriate placeholders
  • Empty placeholders: Remove unused ones or choose a different layout

Quick layout checklist

  • ☐ Used a built-in layout (not blank slide)
  • ☐ Every slide has a unique, descriptive title
  • ☐ Content is in placeholders, not floating text boxes
  • ☐ Reading order flows logically (test with Tab key)
  • ☐ Visual hierarchy is clear and consistent
  • ☐ Layout serves the content, not just visual design
  • ☐ No accessibility checker errors related to structure
