Designing accessible charts and graphs

Charts and graphs are powerful visual communication tools, but they can exclude users who cannot see them or who perceive colors differently. Making them accessible ensures everyone can understand your data story.

Why chart accessibility matters

Consider a pie chart showing market share by region using only colors to distinguish segments. A person who is colorblind or using a screen reader cannot interpret this critical business data without additional accessibility features.

With accessible design: The chart includes patterns for each segment, data labels showing percentages, a clear title, alternative text describing the key findings, and a data table with the exact values.

Creating charts in PowerPoint 365

Using built-in chart tools

  1. Go to Insert > Chart
  2. Choose an appropriate chart type for your data
  3. PowerPoint opens Excel for data entry
  4. Enter your data with clear headers
  5. Close Excel to return to PowerPoint with embedded chart

Advantages of native PowerPoint charts

  • Editable: Data can be updated easily
  • Scalable: Maintain quality at different sizes
  • Screen reader compatible: Better than image-based charts
  • Template consistency: Match your presentation design

Required accessibility features

These are the essential elements that must be included to make your charts and graphs accessible.

Alternative text summarizing chart and graph images

Every chart and graph must include meaningful alternative text that conveys the key information to screen reader users.

What to include in alt text

  • Chart type: Bar chart, line graph, pie chart, etc.
  • Data overview: What data is being presented
  • Key trends: Main patterns or findings
  • Specific values: Important data points

How to add alt text

  1. Right-click on the chart
  2. Select Edit Alt Text
  3. Type your description in the Alt Text pane
  4. Do not check "Mark as decorative" for informational charts

Alt text examples

❌ Poor alt text
  • "Chart"
  • "Graph showing data"
  • "Sales chart"
✅ Good alt text
  • "Bar chart showing quarterly sales from Q1 to Q4 2024. Sales increased steadily from $2.1M in Q1 to $3.4M in Q4, with the largest jump of $800K between Q3 and Q4."
  • "Line chart tracking customer satisfaction from January to December 2024. Satisfaction started at 3.2 stars, dipped to 2.8 in March, then steadily improved to 4.1 stars by year end."

Sufficient color contrast (3:1)

All chart elements must have sufficient color contrast to be visible to users with low vision or color vision deficiencies.

What needs sufficient contrast

  • Chart elements: Minimum 3:1 contrast ratio between adjacent colors (e.g., different bars, pie segments, or lines)
  • Text labels: Minimum 4.5:1 contrast ratio for regular text, 3:1 for large text (18pt or larger)
  • Background separation: Clear distinction between chart and slide background
  • Data markers: Adequate contrast for points, lines, and shapes

How to ensure sufficient contrast

  • Use high contrast color combinations (e.g., dark blue with yellow or orange)
  • Test your charts using a color contrast checker tool
  • Add contrasting borders around chart elements (bars, pie segments)
  • Test your charts in grayscale to verify they remain distinguishable

⚠️ Common contrast issues

  • Pastel colors that are too light against white backgrounds
  • Multiple shades of the same color (e.g., light blue, medium blue, dark blue)
  • Red and green combinations that appear similar to colorblind users
  • Light grey text on white backgrounds

What to avoid

These practices create accessibility barriers and should be avoided when designing charts and graphs.

Relying on color alone to convey meaning

Color should never be the only way to distinguish data in charts and graphs. Many users cannot perceive colors as intended due to colorblindness, low vision, or using assistive technologies.

Why this is a problem

  • Colorblind users: Red-green colorblindness affects approximately 8% of men and 0.5% of women
  • Screen reader users: Cannot perceive color at all
  • Black and white printing: Color distinctions disappear
  • High contrast mode: Colors may be overridden by user settings

Examples of color-only encoding to avoid

  • ❌ Pie chart with only color differences between segments (no labels or patterns)
  • ❌ Line chart with multiple lines distinguished only by color
  • ❌ Bar chart where red means "bad" and green means "good" with no other indicators
  • ❌ Heat maps using only color gradients to show intensity

What to do instead

Always combine color with other visual indicators:

  • Add patterns or textures to chart elements
  • Include direct data labels
  • Use different line styles (solid, dashed, dotted)
  • Vary shapes for data markers
  • Add text annotations for important points

Real-world example of the problem

A quarterly sales chart shows four bars in different colors: red, blue, green, and yellow, representing Q1-Q4. The legend identifies each color with the quarter.

The problem: A colorblind user cannot distinguish the red and green bars. A screen reader user only hears "chart with four bars" without knowing which bar represents which quarter.

The solution: Add patterns to each bar (solid, diagonal stripes, dots, crosshatch), include data labels showing the quarter and value on each bar, and provide descriptive alt text.

Recommended practices

These best practices go beyond the minimum requirements to create charts and graphs that are truly accessible and easy to understand for everyone.

Adding a title for all charts and graphs

Every chart should have a clear, descriptive title that explains what the data shows and helps users understand the purpose of the visualization.

Effective chart titles

  • Descriptive: Explain what the chart shows
  • Include context: Time period, data source, units
  • Highlight key finding: What should viewers take away?
  • Be concise: Keep it brief but informative

Title examples

❌ Weak titles
  • "Sales"
  • "Chart 1"
  • "Data"
✅ Strong titles
  • "Q4 2024 Revenue by Region Shows 23% Growth in Asia-Pacific"
  • "Customer Satisfaction Improved from 3.2 to 4.1 Stars Over Six Months"
  • "Marketing Budget Allocation: Digital Advertising Leads at 45%"

Using patterns to better distinguish colored sections

Patterns and textures provide a redundant way to distinguish chart elements beyond color, making charts accessible to everyone.

How to add patterns in PowerPoint 365

  1. Right-click on a chart element (bar, pie segment, etc.)
  2. Select Format Data Series or Format Data Point
  3. Choose Fill > Pattern fill
  4. Select a distinct pattern for each data series
  5. Adjust foreground and background colors for good contrast

Pattern selection guidelines

  • Use distinctly different patterns: Solid, diagonal stripes, dots, crosshatch, grid
  • Ensure patterns are visible: Test at actual presentation size
  • Maintain consistency: Use the same pattern for the same data across charts
  • Don't overdo it: Too many patterns can be visually overwhelming

Pattern combinations that work well

  • Solid fill
  • Diagonal lines (left-to-right)
  • Dots or small circles
  • Crosshatch (grid)
  • Diagonal lines (right-to-left)

Adding data values next to or inside the chart for clarity

Direct labeling makes data immediately accessible without requiring users to reference axes, legends, or separate tables.

How to add data labels

  1. Click on the chart to select it
  2. Click the + button (Chart Elements) next to the chart
  3. Check Data Labels
  4. Choose label position (center, inside end, outside end)

Data label best practices

  • Include units: Show currency symbols, percentages, or measurement units
  • Format for readability: Use appropriate decimal places
  • Ensure sufficient contrast: Labels must be readable against their background
  • Avoid clutter: For charts with many data points, label only key values
  • Consider placement: Position labels where they don't overlap with chart elements

When to add data labels

  • Simple charts: Always include for bar, column, and pie charts with few elements
  • Key data points: Highlight important values in complex charts
  • Trend points: Label peaks, valleys, and inflection points on line charts
  • Comparisons: Show values when precise comparison is important

Adding an equivalent table of the chart data

Providing the underlying data in a table format ensures that users who cannot see or interpret the chart can still access the information.

When to provide data tables

  • Complex charts: Charts with many data points or multiple series
  • Precise values needed: When exact numbers matter for decision-making
  • Comparative analysis: Side-by-side data comparison required
  • Reference material: Data users might want to reference later

Where to include data tables

  • Same slide: For simple charts with key data (recommended when space permits)
  • Next slide: For supporting detail without cluttering the chart slide
  • Appendix: For comprehensive data sets
  • Speaker notes: For presenter reference
  • Separate handout: Detailed data document for offline access

Creating accessible data tables for charts

  1. Use PowerPoint's Insert > Table feature (never use text boxes)
  2. Include a header row with descriptive column labels
  3. Go to Table Design > Header Row and check the box
  4. Ensure table headers match the chart data categories
  5. Use consistent formatting and alignment
  6. Maintain adequate color contrast (3:1 minimum for large text)

Table structure for different chart types

For bar and column charts:

Quarter Sales ($M)
Q1 2024 2.1
Q2 2024 2.4

For line charts with multiple series:

Month Product A Product B
January 100 85
February 115 90

For pie charts:

Category Percentage Value
Digital Advertising 45% $450,000
Traditional Media 30% $300,000

✅ Complete accessibility example

Chart with all recommended features:

  • Title: "Q4 2024 Revenue by Region Shows 23% Growth in Asia-Pacific"
  • Alt text: "Bar chart showing quarterly revenue by region. Asia-Pacific leads with $3.4M (23% growth), followed by North America at $2.8M (10% growth), Europe at $2.1M (5% growth), and Latin America at $1.2M (15% growth)."
  • Visual design: Each bar uses a different pattern (solid, stripes, dots, crosshatch) in addition to high-contrast colors
  • Data labels: Each bar shows the exact value and percentage growth
  • Data table: Included on the same slide showing all values
  • Contrast: All elements meet 3:1 minimum contrast ratio

Chart types and best practices

Different chart types have specific accessibility considerations. Here are recommendations for the most common chart types used in presentations.

Bar and column charts

Best for: Comparing categories, showing changes over time

Accessibility considerations

  • Use different patterns: Solid, striped, dotted fills for each category
  • Direct labeling: Include values on or near bars
  • Logical ordering: Arrange bars by value or chronological order
  • Adequate spacing: Ensure bars are clearly separated

Alt text example

"Column chart showing 2024 monthly sales. Sales ranged from $180K in January to $320K in December, with steady growth throughout the year and a notable spike to $340K in November."

Line charts

Best for: Showing trends over time, continuous data

Accessibility considerations

  • Different line styles: Solid, dashed, dotted lines for multiple series
  • Distinct markers: Circles, squares, triangles for data points
  • Line thickness variation: Make important lines thicker
  • Data point labels: Key values labeled directly on the chart

Alt text example

"Line chart tracking customer satisfaction from January to December 2024. Satisfaction started at 3.2 stars, dipped to 2.8 in March, then steadily improved to 4.1 stars by year end."

Pie charts

Best for: Showing parts of a whole (use sparingly)

Accessibility considerations

  • Limit segments: 5-7 segments maximum
  • Pattern fills: Different textures for each segment
  • Percentage labels: Include values on each segment
  • Logical ordering: Largest to smallest segments

Alt text example

"Pie chart showing marketing budget allocation: Digital advertising 45%, Traditional media 30%, Events 15%, Content creation 10%."

⚠️ Consider alternatives to pie charts

Pie charts are often difficult to interpret accurately, especially for users with visual impairments. Consider:

  • Horizontal bar charts: Easier to compare values precisely
  • Simple table: More precise for exact values
  • Text list: Clear hierarchy of categories

Scatter plots and combination charts

Best for: Showing relationships between variables, multiple data types

Accessibility considerations

  • Clear axis labels: Describe what each axis represents with units
  • Trend lines: Show patterns or correlations when applicable
  • Key points highlighted: Important data points called out with annotations
  • Alternative summary: Text description of relationships and findings
  • Data table: Essential for complex scatter plots

Charts to avoid or use with caution

  • 3D charts: Distort data perception, harder to read accurately, and reduce contrast
  • Overly complex combinations: Multiple chart types in one can confuse users
  • Too many data series: More than 4-5 lines/categories becomes difficult to distinguish
  • Decorative charts: Prioritize data clarity over visual design

Testing and validation

Accessibility Checker validation

PowerPoint's built-in Accessibility Checker helps identify common issues:

  1. Go to Review > Check Accessibility
  2. Review errors for missing alt text on charts
  3. Address warnings about color contrast
  4. Fix any issues with reading order

Manual testing techniques

Screen reader testing

  1. Use Windows Narrator or another screen reader
  2. Navigate through your charts
  3. Verify alt text is read correctly and conveys key information
  4. Check that data tables are properly announced with headers

Visual accessibility testing

  1. Contrast checking: Use a color contrast analyzer to verify 3:1 minimum for chart elements
  2. Colorblind simulation: Test with colorblindness simulators (many free online tools available)
  3. Grayscale test: Convert your presentation to grayscale to ensure distinctions remain clear
  4. Print test: Print charts in black and white to verify patterns and labels are distinguishable
  5. Zoom testing: Verify readability at 200% zoom level

Keyboard navigation testing

  1. Tab through your presentation using only keyboard
  2. Verify all interactive chart elements are reachable
  3. Check that reading order makes sense

Final checklist for accessible charts and graphs

  • ☐ All charts have meaningful titles
  • ☐ Charts have comprehensive alt text summarizing key findings
  • ☐ Color contrast meets 3:1 minimum for all chart elements
  • ☐ Color is not the only way to distinguish data
  • ☐ Patterns or textures provide redundant encoding
  • ☐ Data labels show values directly on or near chart elements
  • ☐ Data tables provided for complex visualizations (when needed)
  • ☐ Charts work in grayscale/high contrast mode
  • ☐ Text labels meet contrast requirements (4.5:1 or 3:1 for large text)
  • ☐ Reading order is logical
  • ☐ Accessibility Checker shows no critical errors
