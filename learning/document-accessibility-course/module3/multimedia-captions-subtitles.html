Why it matters

They also benefit people in noisy environments, non-native speakers, and those who prefer to read along with audio content.

Multimedia accessibility is essential because:

  • Required by accessibility laws in many jurisdictions
  • Ensures equal access to information for people with hearing disabilities
  • Helps all users understand complex audio content
  • Allows viewing in sound-sensitive environments
  • Assists non-native speakers and language learners

Who benefits from captions?

  • - Primary beneficiaries
  • - Visual text aids comprehension
  • - Multiple input channels improve understanding
  • - Libraries, open offices, public spaces
  • - Reading reinforces audio information

Add captions to embedded video (Windows, Microsoft 365)

Step-by-step process

  1. Insert → Video, select your file
  2. Click on the video to activate video tools
  3. Go to Playback tab → Insert Captions → From File
  4. Browse and add a WebVTT (.vtt) captions file
  5. Play the video to ensure captions display correctly
  6. Use caption settings to position captions appropriately

Requirements for successful caption integration

  • MP4, MOV, WMV work best with PowerPoint
  • WebVTT (.vtt) is preferred
  • Keep caption files in the same folder as your video
  • Captions must be synchronized with audio

Video accessibility beyond captions

  1. and doesn't autoplay
  2. Provide video controls that are keyboard accessible
  3. Include a text description of visual elements if important to understanding
  4. Consider providing a full transcript as an alternative

⚠️ Important limitations

  • Caption files must remain with the video file
  • Some caption features may not work on all devices
  • Online videos may have different caption support
  • Update captions when video content changes

Caption file formats and creation

WebVTT (.vtt) format

WebVTT (Web Video Text Tracks) is the preferred format for PowerPoint captions:

Basic WebVTT structure:

WEBVTT
                  1
                  00:00:00.000 --> 00:00:03.000
                  Welcome to today's presentation on accessibility.

                  2
                  00:00:03.000 --> 00:00:07.000
                  We'll cover key techniques for inclusive design.

Creating caption files

Option 1: Professional captioning services

  • Rev.com, 3Play Media, Verbit- Professional accuracy
  • ClipChamp- Automatic captions for organizational content
  • Cost vs. accuracy trade-offs - Professional services are more accurate

Option 2: AI-powered transcription tools

  • Otter.ai - Good for meetings and presentations
  • Descript - Video editing with integrated transcription
  • Trint - Professional-grade transcription platform
  • Cost vs. Accuracy trade-offs - Professional services are more accurate

Option 3: Manual creation

  • Subtitle Edit - Advanced subtitle editor
  • Aegisub - Advanced subtitle editor
  • Simple text editor - For basic WebVTT files
  • Time-intensive but precise controls

Caption quality standards

  • 99% accuracy for professional content
  • Captions should appear with or slightly before audio
  • Maximum 160-180 words per minute
  • 32-40 characters per line maximum
  • Use >> or names for multiple speakers

Audio-only content accessibility

Transcripts for audio content

Audio-only content (like recorded narration or podcast clips) requires transcripts:

  • Include all spoken content word-for-word
  • Label who is speaking when
  • Note important sounds [applause], [music], [door closing]
  • Include timestamps for longer content

Delivering transcripts

Option 1: Speaker notes

  • Include full transcript in PowerPoint speaker notes
  • Accessible to screen readers and note-sharing
  • Travels with the presentation file

Option 2: Separate document

  • Provide transcript as separate accessible Word document
  • Include link or reference on the slide
  • Easier for users to navigate and search

Option 3: On-slide summary

  • Provide key points directly on the slide
  • Include link to full transcript
  • Good for shorter audio clips

Multimedia Captions and Subtitles checklist

  • ☐ All embedded videos have captions (.vtt, .srt, or .sami files)
  • ☐ Captions are properly synchronized with audio
  • ☐ Captions include speaker identification when needed
  • ☐ Non-speech sounds described in captions [applause], [music]
  • ☐ Caption files tested and verified to load correctly
  • ☐ Audio-only content has complete text transcripts
  • ☐ Transcripts available in accessible format
  • ☐ Caption text has sufficient contrast for readability
