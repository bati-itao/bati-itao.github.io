Using Accessibility Checker in Word

What it is

The Accessibility Checker is a built-in Word tool that automatically scans your document for accessibility issues and provides specific recommendations for fixing them. It identifies problems in three categories:

  • Errors: Content that makes the document very difficult or impossible for people with disabilities to access
  • Warnings: Content that may be difficult for people with disabilities to understand
  • Tips: Suggestions for improving accessibility, though not critical issues

The Accessibility Checker can run in two modes:

  • On-demand: Run manually when you want to check your document
  • Keep checker running: Continuously monitors your document as you work, alerting you to issues in real-time

Why it matters

Using the Accessibility Checker is important because:

  • Catches mistakes early: Identifies accessibility issues while you're creating content, when they're easiest to fix
  • Saves time: Prevents having to go back and fix accessibility issues later
  • Educates as you work: Provides explanations of why issues matter and how to fix them
  • Ensures compliance: Helps meet accessibility standards and organizational requirements
  • Improves quality: Often highlights issues that also affect general usability
  • Real-time feedback: "Keep checker running" option provides immediate notifications

What the Accessibility Checker detects

  • Missing alt text on images
  • Tables without header rows
  • Merged or split cells in tables
  • Poor color contrast
  • Missing document title
  • Hyperlinks with non-descriptive text (like "click here")
  • Manually formatted headings (not using styles)
  • Missing document language
  • Inaccessible content in headers/footers

How to do it

Running the Accessibility Checker

Method 1: From the Review tab

  1. Go to the Review tab in the ribbon
  2. Click Check Accessibility in the Accessibility group
  3. The Accessibility pane will open on the right side showing any issues found

Method 2: From the File menu

  1. Go to File → Info
  2. Click Check for Issues
  3. Select Check Accessibility
  4. Review issues in the Accessibility pane

Enabling "Keep checker running while I work"

This is the most powerful way to use the Accessibility Checker, as it provides real-time feedback:

  1. Open the Accessibility Checker using either method above
  2. In the Accessibility pane, find the option "Keep accessibility checker running while I work"
  3. Toggle this option to On
  4. You'll now see a small accessibility icon in the status bar (bottom of the screen)
  5. The icon shows the number of issues:
    • No icon = No issues
    • Number with icon = Issues detected
  6. Click the icon anytime to open the Accessibility pane and see details

✅ Benefits of "Keep checker running"

  • Immediate feedback when you add inaccessible content
  • Easy to fix issues right away, rather than finding them all later
  • Helps you learn accessible practices through repetition
  • Always know the accessibility status of your document
  • Reduces the time spent on final accessibility review

Understanding and fixing issues

When the Accessibility Checker finds issues:

  1. Review the issue list: Issues are organized by category (Errors, Warnings, Tips)
  2. Click on an issue: This will:
    • Highlight the problematic content in your document
    • Show recommended actions
    • Explain why it matters
    • Provide a "Learn more" link for additional information
  3. Use the recommended actions dropdown: Often provides quick fix options
  4. Fix the issue: Follow the guidance provided
  5. Issue disappears automatically: Once fixed, the issue is removed from the list

Common issues and quick fixes

Issue Quick Fix
Missing alt text Right-click image → Edit Alt Text → Add description
Table missing header row Click table → Table Design → Check "Header Row"
Merged cells Select merged cells → Table Layout → Split Cells, or restructure table
Non-descriptive link text Edit link text to describe the destination
Missing document title File → Info → Properties → Add Title
Manual heading formatting Apply proper Heading styles (Heading 1, 2, 3, etc.)

Best practices for using Accessibility Checker

  • Enable "Keep checker running": Make it a habit to keep this on
  • Fix issues immediately: Address problems as they're detected rather than accumulating them
  • Prioritize errors: Fix all errors first, then warnings, then tips
  • Read the explanations: Understanding why something is an issue helps prevent future mistakes
  • Check before sharing: Always run a final accessibility check before distributing documents
  • Use it as a learning tool: The checker helps you understand accessibility requirements
  • Don't ignore warnings: While not as critical as errors, warnings still impact accessibility

⚠️ Limitations of Accessibility Checker

The Accessibility Checker is helpful but not perfect. It cannot detect:

  • Whether alt text is meaningful and accurate (it only checks if alt text exists)
  • Whether heading text is descriptive
  • Whether long descriptions are needed for complex images
  • Whether links make sense in context
  • All color contrast issues (it checks some but not all)
  • Whether content is written in plain language

Use the Accessibility Checker as one tool in your accessibility toolkit, but also apply your own judgment and understanding of accessibility principles.

Troubleshooting

If the Accessibility Checker button is grayed out:

  • Ensure you're not in editing mode for headers/footers or text boxes
  • Check that you have a document open (not just a blank Word window)
  • Verify your Word version supports the feature (Office 365/2016 or later)

If issues don't disappear after fixing:

  • Click the refresh icon in the Accessibility pane
  • Close and reopen the Accessibility Checker
  • Save and reopen the document

Leveraging accessible Word templates for consistency

What it is

Accessible Word templates are pre-designed document structures that have accessibility features built in from the start. They include:

  • Properly configured heading styles
  • Accessible color schemes with good contrast
  • Pre-formatted table styles with header rows
  • Placeholder text with guidance
  • Correct document properties (title, language, etc.)
  • Accessible page layouts

Templates ensure that every document created from them starts with a strong accessibility foundation, reducing the effort required to create accessible content.

Why it matters

Using accessible templates is important because:

  • Consistency across organization: Everyone uses the same accessible structure and styles
  • Saves time: Don't need to set up accessibility features for each new document
  • Reduces errors: Less chance of forgetting accessibility requirements
  • Easier to maintain: Update the template once, and all new documents benefit
  • Professional appearance: Templates ensure consistent branding and formatting
  • Built-in guidance: Templates can include instructions and examples
  • Promotes best practices: Makes accessible practices the default, not an afterthought

ITAO accessible templates

The IT Accessibility Office (ITAO) provides accessible Word templates specifically designed for Government of Canada use. These templates include:

  • Federal identity program compliant branding
  • Bilingual support
  • Pre-configured accessible styles
  • Guidance for maintaining accessibility
  • Tested for compliance with accessibility standards

How to do it

Finding and using ITAO Word templates

  1. Visit the ITAO website:
    • Navigate to the ITAO resources section (link provided in Resources and Support page)
    • Look for "Accessible Document Templates" or "Word Templates"
  2. Download the appropriate template:
    • Choose based on document type (report, memo, presentation notes, etc.)
    • Select English or bilingual version as needed
  3. Save template to your computer:
    • Save to Word's default template location, or
    • Save to a shared location if your team will use it
  4. Create new documents from the template:
    • File → New → Personal or Custom templates
    • Select the ITAO template
    • Begin adding your content

Using templates effectively

When working with an accessible template:

  • Read the instructions: Most templates include guidance in the document or as comments
  • Use the provided styles: Don't override heading styles or create new ones unnecessarily
  • Replace placeholder text: Remove sample content and add your own
  • Maintain the structure: Don't remove heading styles or table formats
  • Follow embedded guidance: Templates often include tips and reminders
  • Save as a regular document: File → Save As → Word Document (not template format) for your working document

Key features of accessible templates

1. Pre-configured heading styles

  • Heading styles already set up with proper hierarchy
  • Good color contrast and readable fonts
  • Appropriate sizing and spacing
  • Consistent with organizational branding

2. Accessible table styles

  • Tables pre-formatted with header rows
  • Banded rows for readability
  • Good contrast between rows
  • Simple structure (no merged cells)

3. Document properties

  • Document language set correctly
  • Placeholders for title and author
  • Metadata structure ready to fill in

4. Color schemes

  • Color palettes that meet contrast requirements
  • Theme colors you can apply consistently
  • Guidance on maintaining contrast

Creating your own accessible templates

If you need to create a custom template for your team or organization:

  1. Start with an ITAO template or accessible document: Don't start from scratch
  2. Configure heading styles:
    • Modify Heading 1, 2, 3, etc. to match your brand
    • Ensure good contrast and readability
    • Don't change style names or hierarchy levels
  3. Set up table styles:
    • Create or modify table styles with header rows
    • Use banded rows
    • Ensure good contrast
  4. Add placeholder content:
    • Include sample headings, paragraphs, and tables
    • Add instructional comments or notes
    • Provide examples of proper usage
  5. Set document properties:
    • File → Info → Properties
    • Set Title, Author, Company
    • Set document language
  6. Test with Accessibility Checker: Ensure template has no accessibility issues
  7. Save as template:
    • File → Save As
    • Choose file type: Word Template (.dotx)
    • Save to Templates folder or shared location
  8. Document the template: Create a guide explaining how to use it
  9. Get feedback: Have others test the template and suggest improvements

✅ Accessible template checklist

  • ☐ All heading styles use proper hierarchy
  • ☐ Color contrast meets minimum requirements (4.5:1)
  • ☐ Fonts are readable (sans-serif, appropriate sizes)
  • ☐ Table styles include header rows
  • ☐ Document language is set
  • ☐ Document title property is filled in (or has placeholder)
  • ☐ Sample content demonstrates accessible practices
  • ☐ Passes Accessibility Checker with no errors
  • ☐ Instructions or guidance included for users
  • ☐ Consistent with organizational branding

Maintaining and updating templates

Templates need periodic maintenance:

  • Review annually: Check if templates still meet current accessibility standards
  • Gather user feedback: Ask users about problems or needed improvements
  • Update for branding changes: Modify templates when organizational branding updates
  • Test with new Word versions: Ensure templates work with updated software
  • Version control: Keep track of template versions and changes
  • Communicate updates: Let users know when templates are updated and why

Organizational template strategy

For organizations implementing accessible templates:

  • Create a template library: Provide templates for common document types
  • Make templates easy to find: Store in a well-known, accessible location
  • Provide training: Teach staff how to use templates properly
  • Enforce template use: Consider making templates mandatory for certain document types
  • Designate template owners: Assign responsibility for maintaining templates
  • Support and guidance: Provide help resources for template users

Benefits of an organizational template program

  • Consistent accessibility across all documents
  • Reduced training needs (everyone uses same tools)
  • Faster document creation
  • Professional, branded appearance
  • Easier to meet compliance requirements
  • Lower remediation costs
  • Better user experience for people with disabilities
