Adding long descriptions for complex visuals

What it is

While Module 2 covered basic alternative text (alt text) for images, some visuals are too complex to describe adequately in alt text alone. Long descriptions provide detailed explanations of complex images, charts, diagrams, or infographics that convey substantial information.

Complex visuals that may need long descriptions include:

  • Charts and graphs with multiple data series
  • Detailed diagrams showing processes or workflows
  • Complex organizational charts
  • Maps with significant information
  • Infographics combining multiple elements
  • Technical illustrations or schematics
  • Screenshots showing multiple interface elements

A long description typically consists of:

  • Short alt text: Brief summary of the image's purpose (125 characters or less)
  • Long description: Detailed explanation of all important information conveyed by the image

Why it matters

Long descriptions are critical because:

  • Alt text has limits: Screen readers may cut off very long alt text, and lengthy alt text can be overwhelming
  • Complex information needs detail: A chart showing quarterly sales trends across five product lines cannot be adequately described in one sentence
  • Equal access to information: If sighted users gain insights from a complex visual, users who cannot see it need the same information
  • Context and analysis: Long descriptions can include interpretation and context that help users understand the significance of the data
  • Legal requirements: Many accessibility standards require equivalent alternatives for complex images

Example: When you need a long description

Image: A line graph showing monthly sales trends for five product categories over 12 months

Short alt text: "Line graph showing monthly sales by product category for 2024"

Long description: "The graph shows sales trends from January to December 2024 for five product categories: Electronics, Clothing, Home Goods, Books, and Toys. Electronics had the highest sales throughout the year, starting at $50,000 in January and peaking at $85,000 in December. Clothing showed seasonal variation, with lower sales in summer months ($30,000) and peaks in spring ($45,000) and fall ($48,000). Home Goods maintained steady sales around $35,000 monthly. Books and Toys had the lowest sales, ranging between $15,000-$25,000 monthly. All categories showed increased sales in the November-December holiday period."

How to do it

Determining if you need a long description

Ask yourself these questions:

  • Does the image convey data, relationships, or complex information?
  • Would it take more than 1-2 sentences to adequately describe the image?
  • Are there specific data points, trends, or patterns that users need to understand?
  • Would a sighted user gain substantive information from this image?

If you answer "yes" to any of these, consider providing a long description.

Writing effective long descriptions

  1. Start with the big picture: What is the overall purpose or main message of the image?
  2. Describe the structure: What type of visual is it? What are its main components?
  3. Present key information systematically:
    • For charts: Describe trends, comparisons, and notable data points
    • For diagrams: Explain the process flow, relationships, or hierarchy
    • For maps: Describe locations, routes, or geographic relationships
  4. Include important details: Specific data points, patterns, or anomalies
  5. Be objective: Describe what's there, not your interpretation (unless interpretation is the purpose)
  6. Use clear, concise language: Avoid jargon unless necessary

Methods for providing long descriptions in Word

Method 1: Text immediately following the image (Recommended)

The simplest and most accessible approach is to include the detailed description as regular text immediately after the image:

  1. Add the image to your document
  2. Add short alt text to the image (right-click image → Edit Alt Text)
  3. Immediately below the image, add a heading like "Figure 1 Description:" or "Chart Details:"
  4. Write the long description as regular paragraph text
  5. This description benefits all users, not just screen reader users

Method 2: Separate section for descriptions

For documents with many complex images:

  1. In the alt text, reference where the long description can be found: "See detailed description in Appendix A"
  2. Create an appendix section titled "Figure Descriptions" or "Image Descriptions"
  3. List each image with its description: "Figure 3: [detailed description]"
  4. Ensure figures are numbered consistently throughout the document

Method 3: Using captions with descriptions

  1. Add the image to your document
  2. Right-click the image → Insert Caption
  3. In the caption, include both a title and the full description if space allows
  4. Add concise alt text that references the caption: "Chart showing sales data, see caption for details"

✅ Long description best practices

  • Keep short alt text under 125 characters
  • Make long descriptions comprehensive but concise
  • Use headings to structure long descriptions in appendices
  • Include units of measurement and labels from the image
  • Describe colors or visual patterns if they convey meaning
  • Consider providing data tables for complex charts
  • Test descriptions by having someone unfamiliar with the image read them

Alternative: Provide data tables for charts

For charts and graphs, consider providing the underlying data as an accessible table:

  • Create a simple table with the same data shown in the chart
  • Place it immediately after the chart or in an appendix
  • This allows users to understand the exact values and make their own comparisons
  • Ensure the table follows all table accessibility guidelines (header rows, no merged cells, etc.)

Handling Federal Identity Program images (bilingual alt text requirements)

What it is

The Federal Identity Program (FIP) is the visual identity system used by the Government of Canada. It includes official symbols, logos, and branded imagery that appear on government documents, websites, and communications. These images have specific requirements for accessibility, particularly regarding bilingual alt text.

Common FIP images include:

  • Government of Canada wordmark (Canada logo)
  • Departmental logos and signatures
  • Official symbols (Flag, Coat of Arms)
  • Bilingual signs and notices
  • Program or initiative branding

Why it matters

Bilingual alt text for FIP images is important because:

  • Official Languages Act: Federal government communications must be accessible in both English and French
  • Equal access: French-speaking screen reader users must receive the same information as English speakers
  • Brand consistency: Official names and terminology must be accurate in both languages
  • Legal compliance: Failure to provide bilingual alt text can result in accessibility compliance issues
  • Professional standards: Demonstrates commitment to accessibility and official languages requirements

⚠️ Bilingual requirement applies to

  • Any document intended for public distribution
  • Internal communications that may be shared broadly
  • Documents that will be converted to accessible PDFs
  • Templates used by multiple people or departments
  • Training materials and presentations

How to do it

Providing bilingual alt text in Word

Word's alt text field only accepts one language at a time. For bilingual documents or FIP images, use one of these approaches:

Method 1: Both languages in the same alt text field (Recommended for single images)

  1. Right-click the image → Edit Alt Text
  2. In the Alt Text pane, write the description in both languages, clearly separated:

Example:

Government of Canada / Gouvernement du Canada

Or with more detail:

Logo: Government of Canada | Logo : Gouvernement du Canada

Method 2: Language-specific versions of the document

For fully bilingual documents:

  • Create separate English and French versions of the document
  • In each version, provide alt text in the appropriate language
  • Maintain both versions and update them together

Method 3: Caption approach for complex images

  1. Add concise bilingual alt text to the image
  2. Provide a full bilingual caption below the image with detailed description
  3. This makes the description visible to all users

Standard alt text for common FIP images

Use these standard descriptions for common government branding:

Government of Canada Wordmark:

Government of Canada / Gouvernement du Canada

Canadian Flag:

Flag of Canada / Drapeau du Canada

Coat of Arms:

Arms of Canada / Armoiries du Canada

Departmental Signature (example for ESDC):

Employment and Social Development Canada / Emploi et Développement social Canada

Best practices for FIP images

  • Use official translations: Always use the official French name for departments and programs, not machine translations
  • Check with communications: If unsure about the correct bilingual terminology, consult your department's communications or official languages team
  • Be consistent: Use the same alt text every time the same logo or image appears
  • Mark decorative appropriately: If a logo is purely decorative (e.g., watermark), mark it as decorative rather than providing empty alt text
  • Consider image placement: In bilingual documents, consider whether the image needs to appear in both language sections

Resources for official names and translations

  • TERMIUM Plus: Government of Canada terminology database (https://www.btb.gc.ca/tpv2alpha/alpha-eng.html)
  • Federal Identity Program Manual: Official guide to government visual identity
  • Your department's communications team: Can provide approved terminology
  • Translation Bureau: For official translations of new terms or programs

✅ FIP image accessibility checklist

  • ☐ All FIP images have alt text
  • ☐ Alt text includes both official languages where required
  • ☐ Official French translations are used (not machine translations)
  • ☐ Decorative logos are marked as decorative
  • ☐ Alt text is consistent across all instances of the same image
  • ☐ Department and program names use official terminology
  • ☐ Alt text is concise but complete

Special considerations

Bilingual images with text:

If an image contains text in both languages (e.g., a bilingual sign):

  • The alt text should include all visible text in both languages
  • Indicate clearly that the image shows bilingual text
  • Example: Bilingual sign reading "Authorized Personnel Only / Personnel autorisé seulement"

When English version has FIP images but French version doesn't (or vice versa):

  • Each language version should be complete and accessible on its own
  • Don't rely on images in one version to convey information missing in the other
  • Ensure both versions meet accessibility standards independently
