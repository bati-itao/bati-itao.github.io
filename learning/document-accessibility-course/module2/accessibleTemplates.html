Why templates help

Good templates make it easy for anyone to create clear, accessible documents and slides. They save time, prevent common mistakes, and give your team a consistent look and feel.

Keep structure simple

  • Provide a small set of clear heading and body styles. Name them in a way people understand.
  • Use simple layouts. Avoid dense multi-column pages and complex slide designs.
  • Use lists and tables with built‑in styles. Mark table headers in the style, not by hand.
  • Keep the main content in the main flow. Avoid floating text boxes for body text.
  • PowerPoint Templates: Consider locking/protecting selective areas (especially in PowerPoint masters) to discourage layout changes that harm accessibility.

Use readable colours and clear links

  • Choose colours that make text easy to read. Dark text on a light background is a safe choice.
  • Don't use color alone for meaning. Add words or icons too.
  • Make links easy to spot. Use an underline or another strong cue.
  • Avoid putting text on top of busy images. If you must, add a solid or shaded box behind the text.

Placeholders that teach

  • Write sample headings and short paragraphs that show plain language.
  • Show good link text, like “Apply for benefits”, not “click here”.
  • Add short reminders where helpful, such as “Add a short description of the image or mark it decorative”.
  • Avoid filler text like “lorem ipsum”. It does not teach good practice.
  • Usage Instructions: Consider including a first-slide/README usage page with template tips if not already present.

Images, charts, and media

  • Remind authors to add short descriptions for meaningful images, and to mark decorative images when they add no information.
  • Use chart styles with clear labels. Offer a simple table view of the data when possible.
  • Charts and Data: Charts should be paired with data tables for accessibility. See the Tables page for guidance on creating accessible data tables.
  • For audio or video, note that captions or transcripts are required.

Keep templates current

  • Publish templates in one trusted place. Remove old versions.
  • Share what changed when you update a template.
  • Review templates regularly for readability and clarity.

Quick check

  • Can you navigate the template with a keyboard?
  • Are headings, lists, tables, and links easy to understand?
  • Is the text easy to read against the background?
  • Do placeholders explain what “good” looks like?

How to create accessible templates step-by-step

Follow these detailed steps to build templates that help everyone create accessible documents:

Setting up the basic structure

  1. Start with a built-in template:
    • In Word/PowerPoint, go to File → New
    • Choose a simple, clean template from Microsoft's built-in options
    • Avoid complex layouts with multiple columns or decorative elements
  2. Set up heading styles:
    • Go to Home → Styles pane
    • Modify Heading 1, 2, and 3 styles with clear, readable fonts
    • Choose font sizes that create a clear hierarchy (e.g., 18pt, 16pt, 14pt)
    • Use bold formatting and adequate spacing
  3. Create accessible color schemes:
    • Go to Design → Colors
    • Choose or create a color palette with high contrast
    • Test that text is readable against all background colors

Adding helpful placeholder content

  1. Write sample headings:
    • Replace "Heading 1" with examples like "Executive Summary"
    • Use clear, descriptive language that shows good practice
    • Include a note: "[Replace with your section heading]"
  2. Add instructional body text:
    • Write 2-3 sample sentences in plain language
    • Include a note: "[Replace with your content. Keep sentences short and clear.]"
    • Show good link text: "Apply for benefits" not "click here"
  3. Include image placeholders:
    • Insert a simple placeholder image
    • Add sample alt text: "Chart showing quarterly sales data"
    • Include instructions: "[Replace image and write meaningful alt text]"

Building accessible table and list templates

  1. Create a table template:
    • Insert a simple 3x3 table
    • Select the first row and go to Table Design → Header Row
    • Add sample headers like "Item", "Description", "Status"
    • Include a note: "[First row contains column headers]"
  2. Add list examples:
    • Create a bulleted list with 3-4 sample items
    • Show parallel structure: "Review documents", "Submit application", "Receive confirmation"
    • Add a numbered list for step-by-step processes

Setting up accessibility features

  1. Configure language settings:
    • Go to Review → Language → Set Proofing Language
    • Choose the appropriate language for your template
    • Save this as the default
  2. Set up document properties:
    • Go to File → Info → Properties
    • Fill in Title: "[Template Name] - [Year]"
    • Add Subject: "Accessible template for [purpose]"
    • Include keywords related to the template's use
  3. Test with accessibility checker:
    • Go to Review → Check Accessibility
    • Fix any issues that appear
    • Document any remaining warnings with explanations

Saving and distributing your template

  1. Save as template:
    • Go to File → Save As
    • Choose Word Template (.dotx) or PowerPoint Template (.potx)
    • Use a clear, descriptive filename
    • Save to a shared location
  2. Create usage instructions:
    • Write a brief guide explaining the template's features
    • Include accessibility reminders
    • Provide contact information for questions
  3. Test with real users:
    • Ask colleagues to create documents using your template
    • Gather feedback on clarity and usability
    • Update the template based on their input
