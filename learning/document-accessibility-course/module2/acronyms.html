Introduce and expand on first use

Write the full term the first time, followed by the acronym in parentheses. Use the acronym alone afterward.

Example: Web Content Accessibility Guidelines (WCAG). Later: “WCAG requires …”

Consider pronunciation

  • If a screen reader mispronounces a critical acronym, consider adding the full term again.
  • Avoid periods in acronyms (e.g., use “USA,” not “U.S.A.”).

Provide a glossary for long documents

Add a glossary or list of abbreviations for reference, especially in technical documents.

Terminology Consistency: If your project maintains a terminology list, recommend referencing it for consistency across documents.

How to implement acronyms properly step-by-step

Follow these detailed instructions to handle acronyms and abbreviations accessibly in your documents:

First use: Introducing an acronym

  1. Type the full term first:
    • Write out the complete name or phrase
    • Example: "Web Content Accessibility Guidelines"
  2. Add the acronym in parentheses:
    • Follow the full term with the acronym
    • Complete example: "Web Content Accessibility Guidelines (WCAG)"
  3. Use plain language for the full term:
    • Avoid complex jargon in your expansion
    • If needed, add a brief explanation after the acronym

Subsequent uses: After the first reference

  1. Use just the acronym:
    • You can now use "WCAG" alone
    • No need to repeat the full term
  2. Consider context:
    • If it's been many pages, remind readers with the full term again
    • In new sections, you may want to re-introduce important acronyms

In Microsoft Word: Using abbreviation formatting

  1. For basic documents:
    • Simply follow the "full term (acronym)" format
    • This works for most business documents
  2. For formal documents with many acronyms:
    • Create an "Abbreviations" or "Acronyms" section
    • List each acronym with its full form
    • Place this section after your table of contents

Creating an acronym glossary in Word

  1. Add a new section:
    • Create a new page or section titled "Abbreviations" or "List of Acronyms"
    • Place it near the beginning of your document
  2. Format as a simple table:
    • Create a 2-column table
    • Column 1: "Acronym", Column 2: "Full Term"
    • List acronyms alphabetically
  3. Example format:
    • API | Application Programming Interface
    • IT | Information Technology
    • PDF | Portable Document Format
    • WCAG | Web Content Accessibility Guidelines

Special considerations for screen readers

  1. Check pronunciation:
    • Test how screen readers pronounce your acronyms
    • If an acronym is mispronounced badly, consider spelling it out occasionally
  2. Avoid periods in acronyms:
    • Use "USA" instead of "U.S.A."
    • Periods can cause awkward pronunciation
  3. Consider context clues:
    • Even after introducing an acronym, occasionally provide context
    • Example: "The WCAG guidelines recommend..." (reminds readers what WCAG is)

What not to mark as different language

  1. Common acronyms:
    • Don't mark well-known acronyms like NASA, FBI, or CEO
    • These are commonly understood in English
  2. Organization names:
    • Don't mark company acronyms as foreign language
    • Example: BMW, IBM, IKEA are treated as English in business contexts
  3. Technical abbreviations:
    • Standard tech terms like PDF, URL, HTTP don't need language marking
    • They're widely used in English documents

Language Cross-Reference: Acronyms typically should not be marked as a foreign language unless the acronym itself is in a different language and read that way. For more guidance on language settings, see the Language page.

Quick checklist for acronym accessibility

  1. ✓ First use shows full term followed by acronym in parentheses
  2. ✓ Subsequent uses can use acronym alone
  3. ✓ Long documents include an acronym list
  4. ✓ Acronyms are spelled without periods (NASA not N.A.S.A.)
  5. ✓ Context clues help readers remember what acronyms mean
  6. ✓ Common acronyms are not marked as foreign language
