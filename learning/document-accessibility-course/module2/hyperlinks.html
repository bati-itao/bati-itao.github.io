Why descriptive links matter

Hyperlinks connect readers to actions, supplementary information, and documents. Clear, descriptive links help everyone—especially keyboard and screen reader users—understand where a link goes and whether it’s relevant. In Microsoft 365 documents and emails, accessible links support navigation, scanning, and decision-making without forcing the reader to guess a destination.

CAN/ASC – EN 301 549:2024 requires that information and relationships be programmatically determinable and understandable. In practice, that means link purpose should be clear from the text or its surrounding context, and links should be visually perceivable and operable for all users.

How to write link text

  • Make the link text describe the destination or action. Example: “Submit travel expense form” or “IT accessibility policy (PDF)”.
  • Avoid non-descriptive phrases such as “click here,” “more,” “read more,” or bare URLs (https://…).
  • Keep link text concise; use nearby text to provide extra context if needed.
  • If linking to a document, include format or key attributes when helpful: “Quarterly results (Excel)” or “Employee handbook (PDF, 1.2 MB)”.
  • When the same page includes multiple links to similar destinations, vary the link text to differentiate purpose (e.g., “Benefits overview” vs. “Benefits enrollment dates”).

Provide context around links

Many readers navigate documents by tabbing between links or by reviewing a list of links read out of context. Make sure the purpose remains clear even when the link text is isolated.

  • Introduce a link with a short description that explains why the reader should follow it.
  • Avoid making an entire sentence the link; link only the meaningful words.
  • Place links close to the relevant discussion so readers don’t have to search for context.
  • In long documents, provide a short “Related resources” section with descriptive links rather than scattered, repeated links.

Visual distinction and placement

  • Ensure links are visually distinguishable from surrounding text using more than color alone (e.g., underline, weight). This supports users with color vision differences and low vision.
  • Maintain sufficient color contrast for link text. As a general target, normal-size text should meet 4.5:1 contrast against the background. If a non-underlined link relies on color alone, ensure both contrast and a clear hover/focus style.
  • Don’t rely on visual location cues (“the link on the right”). Describe the destination or action.
  • Avoid placing different links too close together, which can make selection difficult for some users.

Link lists, collections, and repeated links

  • Group related links into short, descriptive lists (e.g., “Policies and forms”) rather than embedding many inline links in paragraphs.
  • If several links go to the same destination, consolidate them or keep one primary link per context. Duplicate ambiguous links increase noise for assistive technology users.
  • When linking to different sections of the same destination (e.g., different SharePoint pages), make each link text unique.
  • Avoid consecutive identical links with different destinations (for example, using “Learn more” multiple times with different targets).

Linking to files and documents

  • Use descriptive file names and link text that communicates the document’s purpose. Example: “Project plan – Q4 2025 (Word)”.
  • Include the file type and size when it affects user choice, especially for large documents or formats like PDF.
  • Prefer accessible source formats. If you must link to a PDF, ensure it’s tagged and accessible or provide an accessible alternative.
  • For time-sensitive communications (e.g., emails), link to a single canonical location that will remain valid and up to date.

Common pitfalls and fixes

  • Problem: Using “click here.” Fix: Replace with the action or destination, such as “Complete the security training.”
  • Problem: Bare URLs in body text. Fix: Convert to descriptive links. Reserve bare URLs for technical contexts where copying the address is necessary.
  • Problem: Multiple “Read more” links on the same page. Fix: Make each one unique, such as “Read more about leave entitlements.”
  • Problem: Links differentiated by color alone. Fix: Add underline or another non-color cue, and verify contrast.
  • Problem: Linking entire paragraphs. Fix: Link only the specific words that convey the action or destination.

Quick checks and testing

  • Read only the link texts in order. Do they make sense without surrounding paragraphs?
  • Tab through links to check logical order and focus visibility.
  • Verify contrast and underline/weight are sufficient to distinguish links.
  • In shared documents or emails, ask a colleague to scan the links and predict destinations before clicking.
