What you will learn

You will learn how to write link text that is easy to scan and easy to understand. This helps everyone, especially people who use a keyboard or a screen reader.

Why clear link text matters

Good links tell people exactly what they will get. Clear link text saves time, reduces confusion, and helps people decide if they should follow the link. It also helps people who scan a list of links without the surrounding paragraph.

How to write links

  • Use short, descriptive text that names the action or the destination. Example: “Apply for travel reimbursement”.
  • Avoid “click here”, “read more”, or pasting long web addresses. These do not explain the purpose.
  • Put the link on the words that matter, not on a whole sentence.
  • If the link opens a file, say so when it helps. Example: “Quarterly results (PDF)”.
  • If you have several similar links, make each one specific. Example: “Benefits overview” and “Benefits dates”.

Give helpful context

Place links close to the information they relate to. Add a short line before the link if you need to explain why the link is useful. Avoid writing “the link on the right” or other visual directions.

Linking to files

  • Use clear file names and link text so people know what the file is.
  • Mention the file type and size when it could affect someone’s choice to download.
  • Share the main, up-to-date location when possible, not copies that may go out of date.

Avoid common mistakes

  • Replace “click here” with the action: “Complete security training”.
  • Turn bare URLs into descriptive links. Keep bare URLs only when people need to copy the address.
  • Do not use the same vague link text for different destinations.
  • Make sure links look like links. Use an underline or another clear visual cue.
  • Scannability tip: Avoid linking entire sentences/paragraphs; keep link text concise for better scannability.

Quick check

  • Read the links on the page out loud, one by one. Do they make sense on their own?
  • Tab through the links. Is the order logical? Can you see where focus is?
  • Are links easy to see next to body text?

How to create accessible links step-by-step

Follow these detailed instructions to create links that work well for everyone, including screen reader users:

Writing effective link text

  1. Choose descriptive words:
    • Select 2-5 words that clearly describe the destination
    • Good: "Submit expense report"
    • Avoid: "Click here" or "Read more"
  2. Make links make sense out of context:
    • Screen readers can list all links on a page
    • Each link should be understandable on its own
    • Test: Would the link text make sense in a list?
  3. Include the action or destination:
    • Action examples: "Download", "Apply", "Register", "Contact"
    • Destination examples: "Privacy policy", "User guide", "Contact page"

Creating links in Microsoft Word

  1. Using the Insert Hyperlink dialog:
    • Type your descriptive link text first (e.g., "Employee benefits overview")
    • Select the text
    • Press Ctrl+K (Windows) or Cmd+K (Mac)
    • In the "Address" field, paste or type the URL
    • Click OK
  2. Editing existing link text:
    • Right-click on the link
    • Choose Edit Hyperlink
    • Change the "Text to display" field to be more descriptive
    • Click OK

Creating links in PowerPoint

  1. Adding links to text:
    • Select your descriptive text
    • Go to Insert → Link
    • Choose Insert Link
    • Enter the web address or select a file
    • Click OK
  2. Adding links to shapes or buttons:
    • Right-click on the shape
    • Select Link
    • Add the URL and descriptive text
    • Ensure the shape has accessible text describing its purpose

Linking to files and documents

  1. Include file type and size when helpful:
    • Example: "Annual report 2024 (PDF, 2.5 MB)"
    • Example: "Budget template (Excel, 125 KB)"
    • This helps users decide whether to download
  2. Use clear file names:
    • Link text should match or describe the actual file name
    • Good: "Employee handbook March 2024"
    • Avoid: "Document1" or "Final_version_v3"

Making links visually accessible

  1. Ensure links are underlined:
    • In Word/PowerPoint, go to Home → Font → Underline
    • This helps users distinguish links from regular text
    • Don't use color alone for meaning
  2. Choose accessible link colors:
    • Use your organization's accessible link colors (not necessarily default blue/purple)
    • Ensure underlines are present and contrast meets WCAG 1.4.3/1.4.11
    • Make sure links stand out from surrounding text
  3. Test link visibility:
    • Print preview in black and white
    • Check that links are still distinguishable
    • Ask someone to quickly scan and identify all links

Email links in Outlook

  1. Creating accessible email links:
    • Type descriptive text: "Contact support team"
    • Select the text
    • Press Ctrl+K and enter the email address
    • Don't just paste email addresses as links
  2. Telephone links:
    • Good: "Call customer service: 1-800-555-0123"
    • Link the entire phrase, not just the number
    • Include context about what the number is for

Fixing common link problems

  1. Replace "click here" links:
    • Find: "For more information, click here"
    • Replace with: "Read the complete user guide"
    • Move the descriptive words into the link itself
  2. Fix bare URL links:
    • Don't leave URLs as raw text like "https://example.com/page"
    • Convert to descriptive text: "Company homepage"
    • Keep bare URLs only when people need to copy the address
  3. Avoid duplicate link text:
    • Don't use "Read more" for multiple different links
    • Make each link unique: "Read more about benefits", "Read more about policies"

Testing your links

  1. Screen reader simulation:
    • Read only the link text aloud, ignoring surrounding text
    • Does each link make sense on its own?
    • Can you understand where it will take you?
  2. Visual scan test:
    • Quickly scan the page looking only at blue/underlined text
    • Are all the links easy to spot?
    • Do they stand out from regular text?
  3. Tab navigation test:
    • Use the Tab key to move through all links
    • Is the order logical?
    • Can you see which link has focus?
