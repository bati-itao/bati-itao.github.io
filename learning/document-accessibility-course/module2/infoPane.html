Info Pane: Fill in document details that help everyone

Why this matters

Document details like Title, Author, Language, and Keywords help people find and use your file. Screen readers use this information too. Clear details save time and reduce mistakes when files are shared or exported.

What to fill in

  • Title: A short, clear name that explains what the file is. Avoid codes or internal acronyms.
  • Subject (if available): A brief description or series name.
  • Author/Organization: Use your team or organization when that makes sense.
  • Keywords/Tags: 3–5 words people might search for.
  • Language: Set the main language of the document so spelling and pronunciation are correct.

Keep titles and file names aligned so people can recognize the file without opening it.

Language and mixed content

Set the document’s main language. If a section uses another language for more than a short phrase, mark that section in the file so it is pronounced correctly. Expand acronyms the first time you use them.

Sharing and exporting

Good properties carry into other formats, like PDF. Set the Title and Language before you export. After export, quickly check that the new file shows the right title and language in its properties.

Common fixes

  • Change “Document1” to a clear title.
  • Switch individual names to the team name if that is your standard.
  • Correct the language setting and mark longer passages in other languages.
  • Remove “DRAFT” from the title when the document is final.

Quick check

  • Look at Title, Author/Organization, Language, and Keywords before sharing.
  • Make sure the file name and Title match.
  • Ask a colleague if they can tell what the file is from its Title alone.
