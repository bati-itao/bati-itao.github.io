What it is

Accessible email in Outlook means creating messages that everyone can read and understand, regardless of their abilities or the assistive technology they use. While general accessibility principles apply across all Microsoft 365 applications, Outlook emails require specific techniques due to how they are composed, delivered, and rendered across different email clients.

This module focuses on Outlook-specific accessibility features and techniques that go beyond the general document accessibility principles covered in Module 2. You'll learn to use Outlook's tools effectively to ensure your email communications are inclusive and accessible.

Why it matters

Email is one of the most critical communication tools in the workplace. Creating accessible emails ensures that:

  • Everyone receives your message equally: Recipients using screen readers, voice control, or other assistive technologies can access your content
  • Communication is effective: Well-structured emails are easier for everyone to read and understand
  • Compliance is met: Accessible emails help meet legal requirements and organizational policies
  • Professional standards are maintained: Demonstrates commitment to inclusive communication

Outlook-specific accessibility challenges

Outlook emails face unique accessibility challenges because:

  • Email content is rendered differently across email clients (Outlook, Gmail, mobile apps, etc.)
  • Recipients may have different accessibility settings and assistive technologies
  • Bilingual emails require special navigation structures
  • Limited formatting controls compared to Word or PowerPoint documents
  • HTML rendering varies between email clients

What this module covers

This module provides practical, Outlook-specific guidance on:

  1. Message Format Settings: How to set HTML as your default email format to enable accessibility features
  2. Bilingual Email Structure: Techniques for creating accessible bilingual emails with skip links and proper navigation
  3. Heading Styles: How to apply and modify Outlook's built-in heading styles for proper structure
  4. Data Tables: Best practices for including tables in emails when necessary
  5. Email Templates: Using and creating accessible email templates in Outlook
  6. Support Resources: Where to get help and find additional Outlook-specific accessibility information

What's NOT in this module

The following topics are covered in Module 2 (Global M365 Document Techniques) and apply to all Microsoft 365 applications including Outlook:

  • Color contrast and font basics
  • Alternative text for images
  • Descriptive link text principles
  • Using the Accessibility Checker
  • Abbreviations and proofing language settings

This module focuses exclusively on Outlook-specific techniques and features.
